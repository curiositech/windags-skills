# Component Patterns

This file maps Swiss-modern logic onto common website sections.

## Navigation

### Good pattern
- slim horizontal bar
- exact spacing
- one wordmark
- concise nav items
- one decisive CTA

### Avoid
- oversized pill nav
- floating translucent glass bars
- excessive icon noise

## Hero

### Good pattern
- strong left-aligned or grid-anchored headline
- a short supporting paragraph
- one primary CTA and one secondary option at most
- optional proof or metric block aligned to the same system

### Avoid
- giant marketing gradient as the primary visual event
- three equal CTAs
- explanatory copy that reads like a pitch deck dump

## Feature Sections

### Good pattern
- section title in a narrow column or rail
- supporting modules aligned in repeated widths
- visual change through rhythm, not novelty card treatments

### Avoid
- every feature as a floating rounded rectangle
- icon soup
- alternating layouts with no structural reason

## Pricing

### Good pattern
- typographic contrast does most of the work
- emphasis on one recommended tier through rule, label, accent, or alignment shift
- concise comparison structure

### Avoid
- shiny badge overload
- each plan using a different visual language
- giant pill buttons that dominate the entire section

## Forms

### Good pattern
- simple fields
- strong labels
- precise input rhythm
- minimal ornament

### Avoid
- placeholder-only labels
- soft shadowed input wells
- gratuitous rounded geometry

## Dashboard Surfaces

Swiss-modern dashboard interpretation is usually reductive rather than purist.

### Good pattern
- reduce nested card stacks
- use spacing and thin rules before adding containers
- reserve accent for active state, focus, or critical numbers
- keep data typography consistent

### Avoid
- every metric inside a heavily padded glossy tile
- six semantic colors competing constantly
- visual hierarchy built on decorative background blocks instead of type

## Image Use

### Good pattern
- decisive crops
- disciplined aspect ratios
- captions and metadata aligned
- images used as evidence, atmosphere, or contrast

### Avoid
- image galleries with arbitrary sizes
- decorative stock collage fragments
- soft drop-shadow framing on every asset

## Footer

### Good pattern
- compact, orderly, typographic
- links grouped logically
- legal and utility links handled cleanly

### Avoid
- giant marketing footer that restarts the whole page style

## Tone of Voice

Swiss-modern UI pairs best with copy that is:
- declarative
- specific
- concise
- structured

It pairs poorly with copy that is:
- gushy
- overloaded with adjectives
- filled with playful asides
- dependent on hype language to carry energy
