# Example: Flaky Test Diagnosis

Problem:
- "Figure out why one integration test flakes across CI, local runs, and retries."

Why this is not just workflow:
- specialists update one shared investigation model
- later hypotheses depend on the evolving diagnostic board

Chosen topology:
- planning: `blackboard`
- runtime today: projected `dag` unless specialized dispatch is wired

Board keys:
- evidence
- hypotheses
- suspected-causes
- disproven-causes
- next-probes
